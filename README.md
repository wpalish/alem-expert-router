# Alem Expert Router

**Умное распределение экспертов, менторов и фрилансеров по входящим заявкам** —
с учётом компетенций, загрузки и приоритетов. Рабочий прототип под задачу
Alem School «Автоматизация и оптимизация внутренних процессов».

> Проблема: нет удобного и прозрачного механизма управления заявками и подбора
> исполнителей. Решение: API, который автоматически и **объяснимо** назначает
> лучшего свободного эксперта на каждую заявку.

## Что решает (1:1 с постановкой задачи)

| Требование заказчика | Как закрывается |
|---|---|
| Управление заявками и объявлениями | REST API приёма заявок и пула исполнителей |
| Автоподбор по **компетенциям** | строгая проверка покрытия навыков + оценка глубины владения |
| Учёт **загрузки** | балансировка: ёмкость уменьшается «на лету», никто не перегружен |
| Учёт **приоритетов** | заявки обрабатываются по приоритету, затем по сроку |
| **Прозрачность** управления ресурсами | каждое назначение с объяснением + дашборд загрузки |
| Без роста операционных затрат | полностью автоматический прогон, ноль ручного труда |

## Как работает алгоритм

1. **Жёсткие фильтры** — эксперт недоступен / нет свободной ёмкости / не
   покрывает требуемые навыки → отсеивается.
2. **Оценка 0..100** среди подходящих = взвешенная сумма: компетентность (0.6) +
   балансировка загрузки (0.25) + рейтинг (0.15). Веса настраиваются под
   политику школы без правки кода.
3. **Распределение по приоритету**: высокоприоритетные заявки получают лучших
   свободных исполнителей первыми; ёмкость списывается по ходу.

Каждое назначение возвращает список причин — почему выбран именно этот человек.

## Запуск

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# тесты (доказательство корректности)
pytest -q

# API
uvicorn app.main:app --reload
# Swagger UI: http://localhost:8000/docs
```

Быстрый сценарий:

```bash
curl -X POST localhost:8000/seed       # загрузить демо-данные
curl -X POST localhost:8000/assign     # умное распределение
curl localhost:8000/dashboard          # прозрачная картина загрузки
```

## Архитектура

```
app/
├── models.py     # Pydantic-модели: Expert, Request, Assignment
├── matching.py   # ЯДРО: оценка соответствия + распределение (объяснимое)
├── store.py      # репозиторий (in-memory; легко заменить на PostgreSQL)
├── main.py       # FastAPI: /experts /requests /assign /dashboard
└── seed.py       # демо-данные
tests/
└── test_matching.py   # 8 тестов: компетенции, загрузка, приоритеты, прозрачность
```

## Дорожная карта (MVP → продукт)

- **MVP (есть сейчас):** движок матчинга + REST API + дашборд + тесты.
- **Этап 2:** PostgreSQL, аутентификация (роли: студент / координатор / ментор),
  веб-кабинет подачи заявок, уведомления.
- **Этап 3 (AI-слой):** семантический разбор свободного текста заявки в
  компетенции через LLM/эмбеддинги (заявку можно писать словами — система сама
  извлечёт нужные навыки), прогноз сроков, авто-ребалансировка нагрузки.
